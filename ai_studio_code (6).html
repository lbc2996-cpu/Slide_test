마크다운 슬라이드 제작

코드처럼 관리하는 발표 자료의 실현

안녕하세요. 오늘 우리는 개발자에게 가장 친숙한 언어 중 하나인 마크다운을 사용해 어떻게 효율적으로 발표 자료를 만들 수 있는지 알아볼 것입니다. PPT나 키노트 같은 GUI 도구 없이도 텍스트만으로 깔끔하고 전문적인 슬라이드를 만드는 방법을 배우게 됩니다. 특히 Git으로 버전을 관리하고, 여러 포맷으로 쉽게 변환하는 등 개발 워크플로우에 최적화된 슬라이드 제작법을 익히는 것이 이번 세션의 핵심 목표입니다.

오늘 우리가 배울 내용들

  • Pandoc과 Marp의 특징 비교 및 상황별 선택 기준 정립
  • 실무 활용을 위한 팁과 고급 기능 마스터

이번 시간에는 먼저 왜 마크다운으로 슬라이드를 만들어야 하는지에 대한 큰 그림을 이해하고, 이어서 가장 대표적인 두 가지 도구인 Pandoc과 Marp에 대해 깊이 있게 다룰 것입니다. 각 도구의 기본 사용법부터 시작해서, 실제 업무에서 유용하게 쓸 수 있는 고급 기능까지 차근차근 실습 중심으로 진행하겠습니다. 마지막으로 두 도구를 비교하며 어떤 상황에 어떤 툴이 더 적합한지 판단하는 기준을 제시해 드릴 예정입니다.

왜 마크다운인가?

텍스트 기반 문서화의 장점

  • 버전 관리: Git으로 변경 이력 추적
  • 콘텐츠 집중: 디자인보다 내용 구조화 집중
  • 플랫폼 독립성: 특정 S/W에 비종속
  • 빠른 생산성: 키보드만으로 신속한 작성
  • 재사용성: 블로그, 문서 등으로 쉬운 변환
구분 전통 방식 (PPT) 마크다운 방식
작업 방식 GUI, 마우스 위주 텍스트, 키보드 위주
버전 관리 diff 추적 어려움 Git으로 추적 용이
협업 파일 공유/취합 Pull Request 협업
재사용성 복사/붙여넣기 포맷 변환
많은 분들이 '왜 굳이 마크다운으로 슬라이드를 만들어야 할까?'라고 생각하실 수 있습니다. 가장 큰 장점은 바로 '관리의 용이성'입니다. 슬라이드를 코드처럼 Git으로 관리할 수 있다는 것은, 누가 어떤 내용을 수정했는지 명확히 추적하고, 필요시 이전 버전으로 쉽게 되돌릴 수 있다는 의미입니다. 또한, 디자인에 쏟는 시간을 아껴 콘텐츠의 논리적 흐름을 만드는 데 더 집중할 수 있어 생산성이 크게 향상됩니다.

Pandoc 소개

문서 변환의 스위스 군용 칼

  • Haskell 기반의 다양한 마크업 언어 상호 변환 라이브러리
  • 마크다운을 HTML, LaTeX, Word(docx), PowerPoint(pptx) 등 수많은 포맷으로 변환
  • CLI를 통해 자동화된 문서 변환 파이프라인 구축에 용이
  • 템플릿과 필터 기능으로 고도의 출력물 커스터마이징 지원
graph LR subgraph Input A[Markdown (.md)] end subgraph Tool B(Pandoc) end subgraph Output C[PPTX] D[PDF] E[HTML] end A --> B --> C & D & E
첫 번째로 소개할 도구는 '판독(Pandoc)'입니다. 판독은 흔히 '문서 변환의 스위스 군용 칼'이라고 불릴 만큼 강력한 변환 기능을 자랑합니다. 우리가 작성한 하나의 마크다운 파일을 가지고 파워포인트, PDF, 웹페이지 등 원하는 거의 모든 형태의 문서로 변환할 수 있습니다. 오늘은 이 판독을 활용해 마크다운을 슬라이드로 변환하는 방법에 집중해서 알아보겠습니다.

Pandoc 주요 출력 포맷

하나의 소스로 다양한 결과물 제작

출력 포맷 명령어 (-t) 주요 특징 추천 사용처
PowerPoint pptx 범용적, 편집 용이 비즈니스 발표, 공유용
PDF (Beamer) beamer 수식/인용 강점, 고품질 학술, 논문 발표
reveal.js revealjs 인터랙티브 웹 슬라이드 기술 컨퍼런스, 웹 공유
HTML (Slidy) slidy 가볍고 단순한 웹 슬라이드 간단한 내부 공유
Pandoc의 가장 큰 매력은 바로 이 '원 소스 멀티 유즈(One Source Multi-Use)'입니다. 마크다운 파일 하나만 잘 만들어두면, 내부 공유용으로는 PPTX로, 학회 발표용으로는 수식이 깔끔하게 나오는 Beamer PDF로, 기술 컨퍼런스에서는 멋진 웹 슬라이드인 reveal.js로 자유자재로 변환할 수 있습니다. 각 포맷의 특징을 이해하고 상황에 맞게 활용하는 것이 중요합니다.

Marp 소개

마크다운 프레젠테이션 생태계

  • Marp(Markdown Presentation Ecosystem): 슬라이드 제작 통합 도구
  • VS Code 확장 프로그램: 'Marp for VS Code'가 가장 대중적
  • 실시간 미리보기: 코드 작성과 동시에 결과물 확인
  • 간결한 문법: 직관적인 지시어(directive)로 스타일링
graph TD subgraph Marp A[마크다운] <--> B[실시간 미리보기] B <--> C[쉬운 커스터마이징] A <--> C end D[개발자 친화적] --> A
이제 두 번째 도구인 '마프(Marp)'에 대해 알아보겠습니다. Marp는 이름 그대로 '마크다운 발표 생태계'를 지향하는 프로젝트입니다. 특히 VS Code 에디터와 아주 잘 통합되어, 코드를 짜듯이 슬라이드를 작성하고 실시간으로 결과물을 확인하는 경험을 제공하는 것이 가장 큰 특징입니다. Pandoc이 범용성과 강력한 변환 기능에 초점을 맞춘다면, Marp는 슬라이드 제작 자체의 편의성과 경험에 더 집중한 도구라고 할 수 있습니다.

Marp 주요 기능

테마, 클래스, 그리고 페이지 번호

기능 Marp 지시어 예시 설명
테마 theme: gaia Gaia 테마를 전체 슬라이드에 적용
페이지 번호 paginate: true 슬라이드 우측 하단에 페이지 번호 표시
슬라이드 클래스 <!-- _class: invert --> 현재 슬라이드의 색상을 반전
헤더/푸터 _header: 'My Presentation' 모든 슬라이드에 공통 머리글 추가
Marp는 슬라이드 디자인을 위한 편리한 기능들을 '지시어(directive)' 형태로 제공합니다. 파일 상단에 `theme: gaia`라고 한 줄만 추가하면 전체 디자인이 바뀌고, `paginate: true`를 추가하면 페이지 번호가 생깁니다. 또한, `lead`나 `invert` 같은 미리 정의된 클래스를 주석 안에 넣어주면, 해당 슬라이드의 레이아웃이나 색상을 손쉽게 변경할 수 있습니다. 이는 디자인 작업을 코드로 제어하는 직관적인 경험을 제공합니다.

Marp 이미지 활용

  • 배경 이미지: `![bg](src)`

  • 이미지 필터: `blur`, `grayscale`, `opacity`

## 코드와 함께 보는 개념
- 코드를 왼쪽에 배치
- 개념 설명을 오른쪽에 기술
Marp의 가장 강력한 기능 중 하나는 이미지 처리입니다. 특히 `![bg]` 문법은 매우 유용합니다. 이 문법을 사용하면 이미지를 단순 삽입하는 것을 넘어 슬라이드의 배경으로 설정할 수 있습니다. `left:40%` 와 같은 옵션을 추가하면, 이미지가 슬라이드 왼쪽 40% 영역에만 배경으로 들어가고 나머지 60% 공간에는 텍스트를 배치하는 세련된 레이아웃을 손쉽게 만들 수 있습니다.

Marp 다이어그램 (Mermaid.js)

텍스트로 그리는 다이어그램

  • Mermaid.js 라이브러리 내장 지원
  • `mermaid` 코드 블록 안에 다이어그램 코드 작성
  • 순서도, 시퀀스 다이어그램, 간트 차트 등 지원

예시: 간단한 순서도

graph TD A[사용자 요청] --> B{인증 확인}; B -- 성공 --> C[데이터 처리]; B -- 실패 --> D[오류 반환]; C --> E[결과 응답];
개발자라면 순서도나 아키텍처 다이어그램을 그려야 할 때가 많습니다. Marp는 텍스트로 다이어그램을 그릴 수 있는 Mermaid.js를 지원하기 때문에, 별도의 그림 도구 없이도 슬라이드 안에서 직접 다이어그램을 코드로 작성하고 바로 결과물을 확인할 수 있습니다. 이는 특히 시스템의 흐름이나 프로세스를 설명할 때 매우 강력한 기능입니다.

Marp 노트와 내보내기

공유 및 발표 준비

  • 발표자 노트: 마크다운 주석 `` 안에 기록
  • 발표자 모드: HTML로 내보낸 후 `P` 키로 실행
  • 내보내기 옵션: PDF, HTML, PPTX, 이미지(PNG, JPG)
  • 실행: VS Code Marp 아이콘 > 'Export slide deck...'
graph TD A["Markdown (.md)"] -- Marp --> B[PDF] & C[PPTX] & D[HTML]
Marp의 발표자 노트는 Pandoc과 달리 일반적인 마크다운 주석을 사용해 더 간단합니다. 작성된 슬라이드는 VS Code에서 클릭 몇 번만으로 PDF나 PPTX 파일로 쉽게 변환할 수 있습니다. 특히 PPTX로 내보내면 각 슬라이드가 이미지로 변환되어 들어가기 때문에, 최종 제출 전에 다른 팀원들과 공유하거나 약간의 수정을 가해야 할 때 유용하게 사용할 수 있습니다.

Pandoc vs. Marp

어떤 도구를 선택해야 할까?

Pandoc

'변환'에 초점. 기존 문서를 재활용하거나 다양한 포맷이 필요할 때 유리.

Marp

'제작 경험'에 초점. 슬라이드를 처음부터 만들고 실시간으로 확인할 때 유리.

기준 Pandoc Marp
핵심 기능 문서 포맷 변환 슬라이드 제작
실시간 미리보기 ❌ 불가능 ✅ 가능 (VS Code)
스타일링 템플릿 (복잡) CSS (쉬움)
추천 상황 문서 재활용, Beamer 새 슬라이드 제작, 프로토타이핑
그렇다면 Pandoc과 Marp 중 무엇을 써야 할까요? 정답은 '상황에 따라 다르다' 입니다. 만약 여러분이 이미 작성된 기술 문서를 발표 자료로 빠르게 변환하고 싶거나, LaTeX의 강력한 수식 기능이 필요한 학술 발표를 준비한다면 Pandoc이 좋은 선택입니다. 반면, 아이디어를 빠르게 슬라이드로 만들고, 디자인을 실시간으로 확인하며 작업하고 싶다면 Marp가 훨씬 편리하고 효율적일 것입니다.

실무 활용 팁

마크다운 슬라이드 작성 노하우

  • 버전 관리: 모든 소스를 Git으로 관리
  • 템플릿화: 자주 쓰는 구조, 회사 테마 등 템플릿화

  • 이미지 최적화: 포함 전 이미지 용량 최적화
  • 발표자 노트 활용: 슬라이드는 간결하게, 설명은 노트에

마크다운 슬라이드를 실무에서 제대로 활용하기 위한 몇 가지 팁입니다. 가장 중요한 것은 Git으로 모든 것을 관리하는 습관입니다. 또한, 반복적인 작업을 줄이기 위해 자신만의 Marp 테마나 템플릿을 만들어두면 생산성이 크게 향상됩니다. 슬라이드는 항상 핵심만 담고, 나머지 스토리는 발표자 노트에 담는 것을 잊지 마세요.

심화 학습: reveal.js

웹 기반 프레젠테이션의 끝판왕

  • HTML/CSS/JS로 인터랙티브 슬라이드를 만드는 프레임워크
  • Pandoc으로 마크다운 변환 가능 (`-t revealjs`)
  • 2D 슬라이드, 프래그먼트, 플러그인 등 강력한 기능

기능 비교 인포그래픽

Marp
간편함, 통합 환경
Pandoc+Beamer
학술적, 고품질
Pandoc+reveal.js
인터랙티브, 웹 친화적
조금 더 깊이 들어가고 싶다면 reveal.js를 살펴보는 것을 추천합니다. reveal.js는 웹 기술을 최대한 활용하여 매우 동적이고 화려한 프레젠테이션을 만들 수 있게 해주는 프레임워크입니다. Pandoc을 이용하면 우리가 작성한 마크다운을 reveal.js 슬라이드로 바로 변환할 수 있습니다. 웹 기반으로 발표를 해야 하거나, 청중과 상호작용하는 슬라이드가 필요할 때 최고의 선택이 될 수 있습니다.
